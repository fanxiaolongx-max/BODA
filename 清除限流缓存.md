# 清除限流缓存

如果您遇到"登录尝试次数过多"的提示，有以下几种解决方法：

## 方法1：等待15分钟

限流会在15分钟后自动重置。

## 方法2：重启服务器（推荐）

```bash
# 停止服务器（Ctrl+C）
# 然后重新启动
npm start
```

重启服务器会清除所有限流计数。

## 方法3：清除浏览器缓存

有时浏览器缓存会导致问题：

1. 打开浏览器开发者工具（F12）
2. 右键点击刷新按钮
3. 选择"清空缓存并硬性重新加载"

## 方法4：修改限流配置（开发环境）

如果您是在开发测试阶段，可以临时放宽限制。

编辑 `server.js` 文件，找到：

```javascript
const loginLimiter = rateLimit({
  windowMs: 15 * 60 * 1000,
  max: 10,  // 改成更大的数字，比如 100
  skipSuccessfulRequests: true,
  message: { success: false, message: '登录尝试次数过多，请15分钟后再试' }
});
```

将 `max: 10` 改成 `max: 100` 或更大。

修改后重启服务器。

## 已优化的配置

最新版本已经优化了限流配置：

- ✅ 登录限制从5次提升到10次
- ✅ 成功的登录不计入限制次数
- ✅ 只有失败的登录才会被计数

## 测试建议

开发测试时，建议：

1. 使用正确的账号密码（admin/admin123）
2. 每次测试后等待几秒再重试
3. 如果频繁测试，临时提高限流次数

## 生产环境注意

生产环境请保持适当的限流配置以防止暴力破解：

- 登录限制：10-20次/15分钟
- API限制：100-200次/15分钟
